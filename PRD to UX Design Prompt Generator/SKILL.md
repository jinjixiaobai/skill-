# PRD to UX Design Prompt Generator

P8 级别的 UX 设计需求提取与提示词生成工具。从产品需求文档（PRD）中智能提取设计需求，生成结构化、可执行的设计提示词文档。

## 核心能力

作为 P8 高级体验设计师，本 skill 具备：

1. **需求洞察**：深度理解 PRD 背后的用户场景和业务目标
2. **信息架构**：提取页面结构、导航逻辑、信息层级
3. **交互设计**：识别关键交互流程、状态变化、反馈机制
4. **视觉规范**：定义设计系统、组件规范、视觉层级
5. **细节把控**：覆盖边界场景、异常状态、微交互细节

## 使用场景

- 产品经理提供 PRD，需要快速生成设计需求文档
- 设计评审前，需要结构化的设计提示词作为基准
- 跨团队协作，需要统一的设计语言和规范
- 设计交付前，需要完整的页面清单和设计要点

## 工作流程

### 第 1 步：读取 PRD 文档

```bash
# 通过 URL 或 nodeId 读取 PRD
npx mcporter call dingtalk-docs get_document_content --args '{"nodeId": "<PRD文档URL或ID>"}'
```

**提取重点**：
- 产品目标与核心价值
- 目标用户与使用场景
- 功能列表与优先级
- 页面结构与导航逻辑
- 交互流程与状态变化
- 数据展示与操作反馈

### 第 2 步：分析设计需求

**P8 级别分析框架**：

#### 2.1 信息架构层
- 页面层级关系（首页 → 列表页 → 详情页）
- 导航结构（Tab、侧边栏、面包屑）
- 信息优先级（主要信息、次要信息、辅助信息）

#### 2.2 交互设计层
- 核心任务流程（用户目标 → 操作路径 → 完成反馈）
- 状态管理（加载、成功、失败、空状态）
- 操作反馈（即时反馈、延迟反馈、错误提示）
- 手势与快捷操作

#### 2.3 视觉设计层
- 设计系统（颜色、字体、间距、圆角）
- 组件规范（按钮、输入框、卡片、弹窗）
- 视觉层级（标题、正文、辅助文字）
- 品牌调性（专业、友好、科技感）

#### 2.4 响应式与适配
- 设备适配（移动端、平板、桌面端）
- 屏幕尺寸（375px、768px、1440px）
- 交互方式（触摸、鼠标、键盘）

#### 2.5 边界场景
- 空状态（无数据、无权限、无网络）
- 异常状态（加载失败、操作失败、超时）
- 极限场景（超长文本、大量数据、低网速）

### 第 3 步：生成设计提示词

**输出结构**（Markdown 格式）：

```markdown
# [产品名称] UX 设计提示词

## 一、设计概览

### 1.1 产品定位
- 目标用户：[用户画像]
- 核心价值：[解决什么问题]
- 使用场景：[何时何地使用]

### 1.2 设计原则
- 原则 1：[简洁高效]
- 原则 2：[一致性]
- 原则 3：[可访问性]

### 1.3 设计系统
- 主色调：#[颜色值]（品牌色）
- 辅助色：#[颜色值]（功能色）
- 字体：[字体家族]
- 圆角：[4px/8px/16px]
- 间距：[8px 栅格系统]

---

## 二、页面设计详情

### 页面 1：[页面名称]

**页面目标**：[用户在此页面要完成什么]

**信息架构**：
- 导航栏：[顶部/侧边/底部]，包含 [导航项列表]
- 主要内容区：[布局方式]
  - 区块 1：[内容描述]
  - 区块 2：[内容描述]
- 操作区：[主要操作按钮]

**交互流程**：
1. 用户进入页面 → [展示什么]
2. 用户点击 [元素] → [触发什么]
3. 系统响应 → [反馈什么]

**视觉规范**：
- 标题：[字号/字重/颜色]
- 正文：[字号/行高/颜色]
- 按钮：[尺寸/圆角/颜色/状态]
- 卡片：[阴影/边框/间距]

**状态设计**：
- 加载状态：[骨架屏/加载动画]
- 空状态：[插图/提示文案/引导操作]
- 错误状态：[错误提示/重试按钮]

**响应式适配**：
- 移动端（375px）：[布局调整]
- 平板（768px）：[布局调整]
- 桌面端（1440px）：[布局调整]

**微交互细节**：
- 悬停效果：[颜色变化/阴影]
- 点击反馈：[按压效果/涟漪]
- 过渡动画：[淡入淡出/滑动]

**边界场景**：
- 超长文本：[截断/省略号/展开]
- 无数据：[空状态插图+引导]
- 无权限：[权限提示+申请入口]

---

### 页面 2：[页面名称]

[重复上述结构]

---
```

### 第 4 步：写入钉钉文档

```bash

  "name": "[产品名称] UX 设计提示词",
  "markdown": "<生成的设计提示词内容>"

```

## 提示词生成原则

### P8 级别标准

1. **结构化思维**：清晰的层级结构，便于快速定位
2. **可执行性**：每个设计点都可直接转化为设计稿
3. **完整性**：覆盖所有页面、状态、场景
4. **专业性**：使用设计行业术语，体现专业水准
5. **前瞻性**：考虑扩展性、可维护性、未来迭代

### 细节把控

- **不遗漏**：每个页面都有完整的设计说明
- **不模糊**：具体到像素、颜色值、动画时长
- **不重复**：组件规范统一定义，页面中引用
- **不冗余**：精简表达，避免废话

### 设计语言

- 使用设计系统术语（Design Token、Atomic Design）
- 明确交互模式（Modal、Drawer、Toast）
- 标注视觉规范（字号、行高、间距、颜色）
- 描述动效细节（缓动函数、时长、触发时机）

## 进阶技巧

### 1. 设计系统复用
如果项目已有设计系统，在提示词中引用：
```markdown
视觉规范：参考 [公司设计系统 v2.0]
- 主色调：使用品牌色 Primary-500
- 按钮：使用 Button-Large 组件
```

### 2. 竞品参考
在提示词中加入竞品分析：
```markdown
参考竞品：
- 淘宝：首页信息流布局
- 京东：商品详情页结构
- 拼多多：社交分享机制
```

### 3. 数据驱动
结合数据分析优化设计：
```markdown
设计依据：
- 用户调研：80% 用户希望快速下单
- 数据分析：商品详情页跳出率 45%
- A/B 测试：大按钮转化率提升 12%
```

## 注意事项

1. **PRD 质量**：如果 PRD 不完整，主动提问补充信息
2. **设计权衡**：在提示词中说明设计决策的理由
3. **技术可行性**：考虑前端实现难度，避免过度设计
4. **迭代空间**：为未来功能预留扩展空间
